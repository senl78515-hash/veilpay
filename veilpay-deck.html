02The Problem
On-chain payments are all-or-nothing today.
Every Solana transaction is permanently public. Amounts, senders, and receivers are exposed to anyone — forever.
🏢 DAO pays contributors — everyone sees everyone's salary
📄 Company pays supplier — competitor sees the deal amount
👤 Individual receives large payment — becomes a target
👁
Fully Public
All transaction data permanently visible to anyone
No Privacy
🚫
Fully Hidden (Mixers)
Anonymity that breaks compliance and triggers regulators
No Compliance
The core tension: Transparency makes blockchains trustworthy.
Full transparency makes them unusable for real finance.

03The Insight
Privacy should be programmable.
Not binary.
The right question isn't "should this be hidden?" — it's "who should see this, when, and under what conditions?"
Transaction A Transaction B Transaction C
Public ledger ✓ EXISTS ✓ EXISTS ✓ EXISTS
Public sees [Hidden] [Hidden] [Hidden]

After 90 days $12,000 $5,000 $28,000
This model already exists in traditional finance — bank statements, auditor access, regulatory subpoenas, archival laws.
VeilPay brings it to Solana.

04The Solution
VeilPay — Programmable privacy rules at the transaction layer.
HOW IT WORKS
1

2

3

06Architecture
UI Layer
Send Page  ·  Rules Builder  ·  Audit View  ·  Role Switcher  ·  Next.js + Tailwind + Zustand
● Rule Engine  REAL · IMPLEMENTED
applyRules(fields, mode, rules) → PublicView  ·  Modes: public | private | rule-based  ·  Fields: amount · sender · receiver
⚠ ZK Provider Interface  PLUGGABLE · MOCKED
generateProof(tx, rules) → ZKProof
Current: MockProvider (hash simulation)
Production: Light Protocol ZK Compressed Accounts
● On-Chain Commitment  REAL · LIVE
1. wallet.signMessage(commitment) ← Ed25519 sig
2. Ed25519Program.verify(sig) ← Solana precompile
3. MemoProgram.write(commitment) ← permanent record
● Privacy Policy Engine  REAL · IMPLEMENTED
canReveal(tx, role)  ·  getSecondsUntilReveal(tx)  ·  Roles: public · user · auditor · companyAdmin · daoGovernance
The ZK provider is a pluggable interface. Replacing mock → Light Protocol = zero changes to Rule Engine, Policy Engine, or UI.
